# Кейс: API-документация для Petstore

## Задача

Показать, как может выглядеть компактная API Reference для публичного Swagger Petstore API: с понятной структурой, примерами запросов, вариантами ответов и ссылкой на интерактивную спецификацию.

## Аудитория

- разработчик, который быстро проверяет доступные endpoint-ы;
- тестировщик, которому нужны примеры запросов и ожидаемые ответы;
- технический писатель, который поддерживает API-документацию рядом с OpenAPI-контрактом.

## Подход

- Сначала дать обзор API и текущую версию.
- Разложить endpoint-ы по ресурсам: `Pet` и `Store`.
- Для каждого endpoint-а показать назначение, пример запроса и важные ответы.
- Отдельно дать ссылку на интерактивную спецификацию, чтобы запросы можно было проверить в Swagger UI.

## Структура API Reference

Текущая версия API в кейсе: `v2`.

### Endpoints By Resource

#### Pet

- `GET /pet/findByStatus` - получить питомцев по статусу;
- `GET /pet/{petId}` - получить питомца по ID.

#### Store

- `GET /store/inventory` - получить статистику инвентаря.

## Навигация и проверка

- В статической странице endpoint-ы сгруппированы по ресурсам, чтобы быстро найти нужный сценарий.
- В Swagger UI можно открыть полный контракт, посмотреть параметры и вручную проверить запрос.

## GET /pet/findByStatus

Возвращает массив питомцев по статусу.

### Пример запроса

```bash
curl -X GET "https://petstore.swagger.io/v2/pet/findByStatus?status=available"
```

### Пример ответа (`200 OK`)

```json
[
  {
    "id": 1,
    "name": "doggie",
    "status": "available"
  }
]
```

## GET /pet/{petId}

Возвращает одного питомца по идентификатору.

### Пример запроса

```bash
curl -X GET "https://petstore.swagger.io/v2/pet/1"
```

### Ответы

- `200` - питомец найден;
- `404` - питомец не найден.

## GET /store/inventory

Возвращает количество питомцев по статусам.

### Пример запроса

```bash
curl -X GET "https://petstore.swagger.io/v2/store/inventory"
```
